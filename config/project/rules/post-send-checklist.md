# 返信案作成・送信後チェックリスト（必須）

返信案を作成し、ユーザーが「送信した」「送った」等と報告した時点で、**確認を挟まず即座に**以下を実行する：

1. **カレンダー更新** — 予定の追加・変更・削除があればカレンダーに反映（リスケなら旧イベント削除 + [仮]で新イベント登録）
2. **todo.md 更新** — 予定テーブル + アクション待ちテーブルに反映
3. **relationships.md 更新** — 該当人物のセクションにやり取り履歴追記（サービス事務連絡など該当人物がいない場合はスキップ可）
4. **commit & push** — `cd ~/workspace && git add -A && git commit -m "<変更内容>" && git push`

**「送信した」が後続処理のトリガー。** 返信案の承認で作業を終わらせない。
