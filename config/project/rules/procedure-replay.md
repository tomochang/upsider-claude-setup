# 手順メモリ再実行ルール

## UserPromptSubmit hookからサジェストを受け取ったら

`[PROCEDURE_MATCH]` タグのサジェストがコンテキストに含まれていたら:

1. **指示されたパスの .proc.md を Read ツールで読む**
2. **前提知識セクションを確認** — ツールの制約・罠を把握
3. **パラメータを現在のコンテキストに適応** — {target_user} 等を実際の値に置換
4. **手順に従って実行** — 各Stepの「結果の見方」「失敗パターンと対処」も参照
5. **やってはいけないことセクションを遵守**

## 重要

- 手順は参考情報。状況に合わない場合は適応して良い
- 失敗したら手順を盲目的にリトライせず、失敗パターンの対処法を確認
- 手順にない新しい知見を得たら、セッション中にユーザーに報告（手順改善の材料）

## 手順ファイルの場所

`procedures/` 配下。INDEX.md にインデックスがある。
